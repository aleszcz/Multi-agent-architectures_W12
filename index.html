LangChain's empirical comparison of multi-agent topologies on the same task. This is the "show me the numbers" companion to the conceptual articles above — useful for forming a real intuition about when a supervisor-worker setup beats a flat network, and where multi-agent overhead actually hurts.

Microsoft AutoGen (GitHub)
GitHub

Microsoft's open-source framework for building multi-agent conversations. The main alternative to LangGraph in this space — agents talk to each other in turn-taking conversations rather than walking a graph. Worth reading the README and the examples folder to see how the same multi-agent problem feels different in AutoGen vs. LangGraph.

2. Agent Communication & Protocols
1 resource
A2A Protocol — How Does A2A Work with MCP?
Docs
a2a-protocol.org/latest

The Agent-to-Agent (A2A) protocol spec. Most LangGraph work happens inside one process; A2A is about how independently-built agents discover each other and exchange tasks across the network. The "How does A2A work with MCP?" section is the key read: MCP standardizes how an agent talks to tools/data; A2A standardizes how agents talk to other agents. They sit at different layers and complement each other.

4. Customizing Agents — Skills, Fine-Tuning, Local Models
3 resources
Microsoft's Open-Source SkillOpt (VentureBeat)
News
share.google/oS6K42dp4tBKMAZCr

Coverage of Microsoft's SkillOpt, which upgrades agent "skills" without touching model weights. The interesting wrinkle for an agent-builder: most of your effort goes into the prompts/skills layer rather than the model layer, and there's now tooling that treats those skills as artifacts you can iteratively improve via search/optimization — basically a sibling to DSPy's prompt-optimization approach.

LoRA Fine-Tuning Hyperparameters Guide (Unsloth Docs)
Guide
share.google/H27lSeYiAiW6LKw38

Practical hyperparameter reference for LoRA (Low-Rank Adaptation) fine-tuning. When prompt engineering and RAG have plateaued for your agent and you need actual task-specific weight changes, this is where you start. Covers learning rate, rank (r), alpha, target modules, and dropout — the knobs that actually matter.

How to Run Local AI Models with Hermes Agent (Unsloth)
Guide
share.google/44dLnzmnMZeHhAkmW

Hands-on walkthrough of running Hermes Agent against a locally-served model via Unsloth's tooling. The natural companion to the LoRA guide above — once you've fine-tuned a model, this is how you actually point an agent at it.

After Week 12 — What You Should Be Able to Do

This week is about breadth. You don't need to deeply implement everything here. You should walk away able to make informed choices about the following:

  • Argue for or against splitting an agent into multiple agents on a given task
  • Explain where A2A and MCP each fit, and why they don't compete
  • Sketch a production agent stack — model, orchestration, memory, tools, observability, eval, deploy
  • Know when to reach for LoRA fine-tuning vs. just better prompting/RAG
  • Pick a sandbox strategy before letting an agent execute arbitrary code or open arbitrary URLs
  • Read AutoGen / CrewAI / DSPy code without it feeling foreign relative to LangGraph
